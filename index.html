Big-O Cheat Sheet

Time Complexity — from fastest to slowest:

O(1) — Constant. Array access by index. HashMap lookup. Always instant.
O(log n) — Logarithmic. Binary search. Each step halves the problem. 1 billion → 30 steps.
O(n) — Linear. Loop through array once. Doubles when input doubles.
O(n log n) — Linearithmic. Merge sort, quick sort. Best possible for comparison-based sorting.
O(n²) — Quadratic. Nested loops. Bubble sort. Avoid for large inputs.
O(2ⁿ) — Exponential. Recursion without memoization. Fibonacci (naive). Blows up instantly.
O(n!) — Factorial. Permutations. 20! = 2.4 quintillion. Only for tiny inputs.

Space complexity = extra memory your algorithm uses (ignoring input). O(1) = no extra memory. O(n) = array proportional to input.
🎯 Interview rule of thumb: n ≤ 20 → O(2ⁿ) ok · n ≤ 500 → O(n²) ok · n ≤ 10⁶ → O(n log n) or O(n) needed · n > 10⁷ → must be O(log n) or O(1)

Arrays & Strings
DSA → Arrays & Strings

Theory — What is an Array?

An array is a collection of elements stored at contiguous memory locations. In Python the built-in list acts as a dynamic array.

Why arrays? Random access is O(1) — you can jump to index 5 instantly without scanning from index 0. This is the killer feature.

Time complexities:
Access by index → O(1)
Search (unsorted) → O(n)
Insert at end → O(1) amortised
Insert at middle → O(n) (must shift elements)
Delete → O(n)

Two key patterns you MUST know:
1. Two Pointers — left & right pointers moving toward each other. Used for: palindrome, pair sum in sorted array, trapping rain water.
2. Sliding Window — a window of fixed or variable size slides across the array. Used for: max subarray, longest substring without repeat.
PYTHON # Array basics arr = [10, 20, 30, 40, 50] print(arr[0]) # 10 → O(1) access print(arr[-1]) # 50 → last element print(arr[1:4]) # [20, 30, 40] → slicing O(k) print(arr[::-1]) # [50,40,30,20,10] → reverse arr.append(60) # O(1) add to end arr.insert(2, 99) # O(n) insert at index 2 arr.pop() # O(1) remove last arr.pop(0) # O(n) remove first (shifts all!)
💡 Memory trick: arr[i] = base_address + i × element_size. That formula is why access is O(1). The CPU computes the address directly — it doesn't loop.

Pattern 1 — Two Pointers

Two pointers start at opposite ends and move inward. Turns O(n²) brute force into O(n). Classic use: check if sorted array has a pair that sums to target.
PYTHON # Two Sum II — sorted array version (classic interview Q) def two_sum_sorted(arr, target): left, right = 0, len(arr) - 1 while left < right: s = arr[left] + arr[right] if s == target: return [left, right] # found! elif s < target: left += 1 # sum too small → move left up else: right -= 1 # sum too big → move right down return [] print(two_sum_sorted([1,3,5,7,9], 12)) # [2, 4] → arr[2]+arr[4] = 5+7 = 12 # Palindrome check using two pointers def is_palindrome(s): l, r = 0, len(s) - 1 while l < r: if s[l] != s[r]: return False l += 1; r -= 1 return True print(is_palindrome("racecar")) # True print(is_palindrome("hello")) # False

Pattern 2 — Sliding Window

A window is a subarray of fixed or variable size. You slide it across instead of recomputing from scratch each time. Turns O(n²) into O(n).
PYTHON # Max sum of k consecutive elements (Fixed window) def max_subarray_sum(arr, k): window_sum = sum(arr[:k]) # first window max_sum = window_sum for i in range(k, len(arr)): window_sum += arr[i] - arr[i - k] # add new, remove old max_sum = max(max_sum, window_sum) return max_sum print(max_subarray_sum([2,1,5,1,3,2], 3)) # 9 (5+1+3) # Longest substring without repeating characters (Variable window) def length_of_longest_substring(s): seen = set() left = max_len = 0 for right in range(len(s)): while s[right] in seen: seen.remove(s[left]) left += 1 seen.add(s[right]) max_len = max(max_len, right - left + 1) return max_len print(length_of_longest_substring("abcabcbb")) # 3 ("abc") print(length_of_longest_substring("pwwkew")) # 3 ("wke")
🎯 Sliding window trigger words in interviews: "subarray/substring", "consecutive", "maximum/minimum length", "contains". When you see these, think sliding window first.

Linked Lists
DSA → Linked Lists

Theory — What is a Linked List?

A linked list is a chain of nodes. Each node holds data + a pointer to the next node. Unlike arrays, elements are not stored contiguously in memory.

Array vs Linked List recap:
Access: Array O(1) vs LL O(n) — must traverse from head
Insert/Delete at head: Array O(n) vs LL O(1)
Insert/Delete at middle: Both O(n) — LL wins on the actual pointer update (O(1)) but finding the spot is O(n)
Memory: Array is cache-friendly. LL has overhead (pointer per node).

Types:
• Singly — each node points to next only
• Doubly — each node points to next AND prev
• Circular — last node points back to head
PYTHON # Node class — the building block class Node: def __init__(self, data): self.data = data self.next = None # points to next node # Build: 1 → 2 → 3 → None head = Node(1) head.next = Node(2) head.next.next = Node(3) # Traverse — always use a temp pointer, never move head! def print_list(head): cur = head while cur: print(cur.data, end=" → ") cur = cur.next print("None") print_list(head) # 1 → 2 → 3 → None
💡 Golden rule: Never move head directly while traversing — you'll lose the list. Always create a temp pointer: cur = head and move cur.

Core Operations

PYTHON # Insert at head — O(1) def insert_head(head, val): new_node = Node(val) new_node.next = head # new node points to old head return new_node # new head # Insert at tail — O(n) def insert_tail(head, val): new_node = Node(val) if not head: return new_node cur = head while cur.next: # go to last node cur = cur.next cur.next = new_node return head # Delete a node by value — O(n) def delete_node(head, val): if head.data == val: return head.next # delete head cur = head while cur.next and cur.next.data != val: cur = cur.next if cur.next: cur.next = cur.next.next # skip the node return head

02
Find Middle of Linked List
classic
Fast & slow pointer (Floyd's algorithm). Slow moves 1 step, fast moves 2. When fast hits end, slow is at middle.
SOLUTION def find_middle(head): slow = fast = head while fast and fast.next: slow = slow.next # 1 step fast = fast.next.next # 2 steps return slow # slow is at middle # 1→2→3→4→5: slow lands on 3 (middle) # 1→2→3→4: slow lands on 3 (second middle)
03
Detect Cycle (Floyd's Cycle Detection)
interview fav
If there's a cycle, fast pointer eventually catches slow. If no cycle, fast hits None.
SOLUTION def has_cycle(head): slow = fast = head while fast and fast.next: slow = slow.next fast = fast.next.next if slow == fast: # they meet → cycle exists return True return False # fast hit None → no cycle

Stacks & Queues
DSA → Stacks & Queues

Theory

Stack — LIFO (Last In First Out)
Like a stack of plates. You add to the top, remove from the top. Used in: function call stack, undo/redo, balanced parentheses, browser back button.

Queue — FIFO (First In First Out)
Like a queue at a shop. First person in = first person served. Used in: BFS, task scheduling, printer queue, messaging systems.

Deque (Double-Ended Queue) — can add/remove from both ends. O(1) all operations. Used for sliding window maximum.
PYTHON # Stack using list stack = [] stack.append(1) # push → O(1) stack.append(2) stack.append(3) print(stack[-1]) # peek → 3 (top of stack) stack.pop() # pop → O(1) removes 3 print(stack) # [1, 2] # Queue using collections.deque (NOT list — list.pop(0) is O(n)!) from collections import deque queue = deque() queue.append(1) # enqueue → O(1) queue.append(2) queue.append(3) queue.popleft() # dequeue → O(1) removes 1 print(queue) # deque([2, 3])
⚠️ Common mistake: Using list.pop(0) for queue — this is O(n) because it shifts all elements. Always use collections.deque for queues. deque.popleft() is O(1).

01
Valid Parentheses
classic
Check if brackets are balanced. Stack-based. Every opener goes on stack, every closer must match top of stack.
SOLUTION def is_valid(s): stack = [] mapping = {")": "(", "}": "{", "]": "["} for ch in s: if ch in mapping: # it's a closer top = stack.pop() if stack else "#" if mapping[ch] != top: return False else: stack.append(ch) # it's an opener return not stack # stack must be empty print(is_valid("({[]})")) # True print(is_valid("([)]")) # False

monotonic stack
For each element, find the next element to its right that is greater. Use a monotonic stack.
SOLUTION def next_greater(arr): result = [-1] * len(arr) # -1 if no greater element stack = [] # stores indices for i in range(len(arr)): while stack and arr[i] > arr[stack[-1]]: idx = stack.pop() result[idx] = arr[i] # i is the next greater for idx stack.append(i) return result print(next_greater([4,1,2,3])) # [-1, 2, 3, -1]

Recursion & Backtracking
DSA → Recursion & Backtracking

Theory — Recursion

Recursion = a function calling itself. Every recursive function has exactly two parts:

1. Base case — when to stop. Without this, you get infinite recursion → stack overflow.
2. Recursive case — the function calls itself with a smaller version of the problem.

The call stack: Each function call is pushed onto the call stack. When base case is hit, calls return and pop off the stack. This is why deep recursion can cause stack overflow — too many frames.

Backtracking = recursion + undoing choices. Try a path → if it fails → undo (backtrack) → try next path. Used in: N-Queens, Sudoku, generating permutations/subsets.
PYTHON # Classic: Fibonacci with and without memoization def fib_naive(n): # O(2ⁿ) — TERRIBLE if n <= 1: return n return fib_naive(n-1) + fib_naive(n-2) # Memoization — store results → O(n) memo = {} def fib(n): if n in memo: return memo[n] if n <= 1: return n memo[n] = fib(n-1) + fib(n-2) return memo[n] # Power function O(log n) using recursion def power(base, exp): if exp == 0: return 1 # base case if exp % 2 == 0: # even: x^n = (x^(n/2))^2 half = power(base, exp//2) return half * half return base * power(base, exp-1) print(power(2, 10)) # 1024

Backtracking — Subsets & Permutations

PYTHON # Generate ALL subsets of [1,2,3] def subsets(nums): result = [] def backtrack(start, current): result.append(current[:]) # add copy of current for i in range(start, len(nums)): current.append(nums[i]) # choose backtrack(i+1, current) # explore current.pop() # un-choose (backtrack!) backtrack(0, []) return result print(subsets([1,2,3])) # [[], [1], [1,2], [1,2,3], [1,3], [2], [2,3], [3]] # Generate ALL permutations def permutations(nums): result = [] def backtrack(path, used): if len(path) == len(nums): result.append(path[:]) return for i in range(len(nums)): if used[i]: continue used[i] = True path.append(nums[i]) backtrack(path, used) path.pop() used[i] = False backtrack([], [False]*len(nums)) return result print(permutations([1,2,3])) # 6 permutations
🎯 Backtracking template: choose → explore → un-choose. The un-choose step (pop/undo) is what makes it backtracking. Without it, you'd carry over state from previous branches.

Cross Questions

Q1
What's the difference between recursion and iteration?
theory
Answer: Both can solve the same problems. Recursion uses the call stack (implicit). Iteration uses a loop (explicit). Recursion is cleaner for tree/graph problems, divide & conquer. Iteration is more memory-efficient (no stack overhead). Tail recursion can be optimized to iteration by compilers.
Q2
Why does naive Fibonacci have O(2ⁿ) complexity?
theory
Answer: fib(5) calls fib(4) and fib(3). fib(4) calls fib(3) and fib(2). fib(3) is computed TWICE. This redundancy grows exponentially — the same subproblems are solved over and over. Memoization fixes this by caching results, turning it to O(n).

Binary Search
DSA → Binary Search

Theory

Binary search works on sorted arrays. It cuts the search space in half each step → O(log n).

Real example: 1 billion elements. Linear search = up to 1,000,000,000 comparisons. Binary search = up to 30 comparisons. That's the power of log₂(10⁹) ≈ 30.

The 3-pointer template (memorise this):
left = 0, right = len(arr) - 1
while left ≤ right:
  mid = left + (right - left) // 2  ← avoids integer overflow
  if arr[mid] == target: found!
  elif arr[mid] < target: left = mid + 1  ← search right half
  else: right = mid - 1  ← search left half
PYTHON # Standard binary search def binary_search(arr, target): left, right = 0, len(arr) - 1 while left <= right: mid = left + (right - left) // 2 # safe mid calculation if arr[mid] == target: return mid elif arr[mid] < target: left = mid + 1 else: right = mid - 1 return -1 # not found arr = [1, 3, 5, 7, 9, 11, 13] print(binary_search(arr, 7)) # 3 (index) print(binary_search(arr, 6)) # -1 (not found)

Variants — Must Know

PYTHON # Find first occurrence of target def first_occurrence(arr, target): left, right, result = 0, len(arr)-1, -1 while left <= right: mid = (left + right) // 2 if arr[mid] == target: result = mid # save, but keep searching LEFT right = mid - 1 elif arr[mid] < target: left = mid + 1 else: right = mid - 1 return result print(first_occurrence([1,2,2,2,3], 2)) # 1 # Search in Rotated Sorted Array (classic interview) def search_rotated(arr, target): left, right = 0, len(arr) - 1 while left <= right: mid = (left + right) // 2 if arr[mid] == target: return mid if arr[left] <= arr[mid]: # left half is sorted if arr[left] <= target < arr[mid]: right = mid - 1 else: left = mid + 1 else: # right half is sorted if arr[mid] < target <= arr[right]: left = mid + 1 else: right = mid - 1 return -1 print(search_rotated([4,5,6,7,0,1,2], 0)) # 4
💡 Use mid = left + (right - left) // 2 instead of (left + right) // 2. In Python integers don't overflow, but in Java/C++ (left + right) can overflow when both are large. Practice the safe form now.

Cross Questions

Q1
Why left ≤ right and not left < right?
gotcha
Answer: When left == right, there's still one element to check. If you use <, you skip the last element. Use ≤ so the single-element case is handled. Switch to < only in specific variants where you want to stop before convergence.
Q2
Find square root of N (without sqrt) using binary search
creative
SOLUTION def sqrt_floor(n): if n < 2: return n left, right = 1, n // 2 while left <= right: mid = (left + right) // 2 if mid * mid == n: return mid elif mid * mid < n: left = mid + 1 else: right = mid - 1 return right # floor of sqrt print(sqrt_floor(25)) # 5 print(sqrt_floor(10)) # 3 (floor of 3.16)

Sorting Algorithms
DSA → Sorting

Theory — Why Learn Sorting?

Sorting is the foundation of efficiency. Binary search requires sorted input. Many greedy algorithms sort first. Interviewers love sorting because it reveals whether you understand complexity trade-offs.

Complexity summary:
Bubble Sort → O(n²) time, O(1) space — for teaching only
Selection Sort → O(n²) time, O(1) space — for teaching only
Insertion Sort → O(n²) worst, O(n) best — good for nearly sorted data
Merge Sort → O(n log n) always, O(n) space — stable, predictable
Quick Sort → O(n log n) average, O(n²) worst, O(log n) space — fastest in practice
Python's sort() → Timsort, O(n log n) — uses merge + insertion sort hybrid

Merge Sort — Divide & Conquer

Split array in half recursively until size 1, then merge back in sorted order. Always O(n log n). Stable (equal elements keep relative order).
PYTHON def merge_sort(arr): if len(arr) <= 1: return arr # base case: already sorted mid = len(arr) // 2 left = merge_sort(arr[:mid]) # sort left half right = merge_sort(arr[mid:]) # sort right half return merge(left, right) # merge them def merge(left, right): result = [] i = j = 0 while i < len(left) and j < len(right): if left[i] <= right[j]: result.append(left[i]); i += 1 else: result.append(right[j]); j += 1 result.extend(left[i:]) # leftover from left result.extend(right[j:]) # leftover from right return result print(merge_sort([38,27,43,3,9,82,10])) # [3, 9, 10, 27, 38, 43, 82]

Quick Sort — Partition & Conquer

PYTHON def quick_sort(arr): if len(arr) <= 1: return arr pivot = arr[len(arr) // 2] # pick middle as pivot left = [x for x in arr if x < pivot] # smaller than pivot mid = [x for x in arr if x == pivot] # equal to pivot right = [x for x in arr if x > pivot] # larger than pivot return quick_sort(left) + mid + quick_sort(right) print(quick_sort([3,6,8,10,1,2,1])) # [1, 1, 2, 3, 6, 8, 10]
💡 Quick sort vs Merge sort: Quick sort is faster in practice (better cache locality, less memory). Merge sort is better when you need guaranteed O(n log n) or stability. Python's built-in sort is Timsort — never re-implement unless asked in an interview.

theory

Trees & BST
DSA → Trees & Binary Search Trees

Theory — Tree Terminology

A tree is a hierarchical data structure. Unlike arrays/linked lists (linear), trees branch.

Key terms:
Root — top node, has no parent
Leaf — node with no children
Height — longest path from root to any leaf
Depth — distance from root to a specific node
Binary Tree — each node has at most 2 children (left, right)
BST (Binary Search Tree) — left < root < right for every node. Gives O(log n) search.
Balanced BST — height is O(log n). Degenerate (all right) = O(n) height = linked list.

Theory — Graphs

A graph is a set of vertices (nodes) connected by edges. Trees are a special case of graphs (no cycles).

Types:

BFS — Breadth First Search

Explores level by level. Uses a queue. Finds shortest path in unweighted graphs. Time: O(V+E).
PYTHON from collections import deque def bfs(graph, start): visited = set([start]) queue = deque([start]) order = [] while queue: node = queue.popleft() order.append(node) for neighbor in graph[node]: if neighbor not in visited: visited.add(neighbor) queue.append(neighbor) return order print(bfs(graph, 0)) # [0, 1, 2, 3, 4] # BFS Shortest Path (unweighted) def shortest_path(graph, start, end): visited = {start} queue = deque([(start, [start])]) while queue: node, path = queue.popleft() if node == end: return path for neighbor in graph[node]: if neighbor not in visited: visited.add(neighbor) queue.append((neighbor, path + [neighbor])) return [] print(shortest_path(graph, 0, 3)) # [0, 1, 3]

DFS — Depth First Search

Goes deep before wide. Uses recursion (or explicit stack). Used for: cycle detection, topological sort, connected components, maze solving.
PYTHON # DFS recursive def dfs(graph, node, visited=None): if visited is None: visited = set() visited.add(node) print(node, end=" ") for neighbor in graph[node]: if neighbor not in visited: dfs(graph, neighbor, visited) dfs(graph, 0) # 0 1 3 2 4 # Count connected components def count_components(graph): visited = set() count = 0 for node in graph: if node not in visited: dfs(graph, node, visited) count += 1 # each new DFS = new component return count

Dijkstra's Algorithm — Weighted Shortest Path

Finds shortest path in weighted graphs with non-negative weights. Uses a min-heap (priority queue). Time: O((V+E) log V).
PYTHON import heapq def dijkstra(graph, start): # graph: {node: [(neighbor, weight), ...]} dist = {node: float('inf') for node in graph} dist[start] = 0 heap = [(0, start)] # (distance, node) while heap: d, u = heapq.heappop(heap) if d > dist[u]: continue # outdated entry for v, w in graph[u]: if dist[u] + w < dist[v]: dist[v] = dist[u] + w heapq.heappush(heap, (dist[v], v)) return dist # Weighted graph wgraph = {0: [(1,4),(2,1)], 1: [(3,1)], 2: [(1,2),(3,5)], 3: []} print(dijkstra(wgraph, 0)) # {0:0, 1:3, 2:1, 3:4}
🎯 Dijkstra fails on negative weights. Use Bellman-Ford for negative edges. Dijkstra is greedy — once a node is finalized (popped from heap), its distance won't change. Negative edges break this guarantee.

Recursion
Python → Recursion

Theory

What is Recursion?
Recursion is when a function calls itself. Instead of using a loop to repeat something, the function keeps calling itself with a smaller version of the problem — until it hits a base case that stops it.

Every recursive function needs TWO things:
1. Base case — the condition that STOPS the recursion (without this = infinite loop / crash)
2. Recursive case — the function calling itself with a simpler/smaller input
STRUCTURE OF RECURSION def my_function(n): if n == 0: # BASE CASE — stop here! return 1 return n * my_function(n - 1) # RECURSIVE CASE
CLASSIC EXAMPLE — FACTORIAL def factorial(n): if n == 0 or n == 1: # base case return 1 return n * factorial(n - 1) # calls itself! # How it runs for factorial(4): # factorial(4) → 4 * factorial(3) # factorial(3) → 3 * factorial(2) # factorial(2) → 2 * factorial(1) # factorial(1) → 1 ← base case hit! # Answer builds back up: 2*1=2, 3*2=6, 4*6=24
FIBONACCI WITH RECURSION def fib(n): if n <= 1: # base case return n return fib(n-1) + fib(n-2) # two recursive calls print(fib(6)) # 8 (0,1,1,2,3,5,8...)
⚠️ Always write the BASE CASE first — before writing the recursive call. If you forget it, Python will crash with "RecursionError: maximum recursion depth exceeded".
💡 Recursion vs Loop: Recursion is cleaner for problems that have a naturally "nested" or "tree-like" structure (like file folders, binary search, tree traversal in DSA). For simple counting, a loop is usually better.

Mean, Median & Mode

These are the three ways to describe the "centre" of a dataset. Each tells you something different — knowing which to use is what separates good data scientists.

Mean — the average. Add all values, divide by count. Sensitive to outliers.
Median — the middle value when sorted. If even number of values, average the two middle ones. NOT affected by outliers.
Mode — the most frequent value. A dataset can have no mode, one mode, or multiple modes.
PYTHON import numpy as np data = [10, 20, 20, 30, 40, 50, 1000] # 1000 is an outlier mean = np.mean(data) # 167.1 ← dragged up by outlier! median = np.median(data) # 30 ← not affected by outlier ✅ # Mode — from scratch (numpy has no direct mode) from collections import Counter mode = Counter(data).most_common(1)[0][0] # 20 print(f"Mean: {mean}, Median: {median}, Mode: {mode}")
🎯 ML interview question: "When would you use median over mean?" → Always say: when data has outliers or is skewed (salaries, house prices, user ages). This question is asked constantly.

Variance & Standard Deviation

Mean tells you the centre. Variance and std dev tell you how spread out the data is around that centre.

Variance — average of squared differences from the mean. High variance = data is spread far from the mean.
Standard Deviation — square root of variance. Same unit as the original data, easier to interpret.

Why square the differences? Because some differences are negative (below mean) and some positive (above mean). Squaring makes them all positive so they don't cancel out.
PYTHON — FROM SCRATCH data = [10, 20, 30, 40, 50] mean = sum(data) / len(data) # 30 diffs = [x - mean for x in data] # [-20, -10, 0, 10, 20] squared = [d ** 2 for d in diffs] # [400, 100, 0, 100, 400] variance = sum(squared) / len(squared) # 200 std_dev = variance ** 0.5 # 14.14 print(f"Variance: {variance}, Std Dev: {std_dev:.2f}")
PYTHON — WITH NUMPY import numpy as np data = [10, 20, 30, 40, 50] print(np.var(data)) # 200.0 print(np.std(data)) # 14.14
💡 Low std dev = data is consistent. High std dev = data is noisy/spread out. In ML, features with very different std devs need StandardScaler before training — otherwise the model gives unfair importance to features with larger values.

Normal Distribution

The Normal Distribution (bell curve / Gaussian) is the most important distribution in statistics and ML. Data clusters around a central mean with symmetrical tails on both sides.

The 68-95-99.7 Rule:
68% of data falls within 1 std dev from the mean
95% of data falls within 2 std devs from the mean
99.7% of data falls within 3 std devs from the mean

Data points outside 3 std devs are considered outliers.
PYTHON — PLOT BELL CURVE import numpy as np import matplotlib.pyplot as plt data = np.random.normal(loc=0, scale=1, size=1000) # mean=0, std=1 plt.hist(data, bins=30, color="#6ec4f5", edgecolor="black", density=True) plt.axvline(0, color="red", linewidth=2, label="Mean (0)") plt.axvline(1, color="orange", linewidth=1, linestyle="--", label="+1 std dev") plt.axvline(-1, color="orange", linewidth=1, linestyle="--", label="-1 std dev") plt.title("Normal Distribution") plt.legend() plt.show()
🎯 In ML: always plot a histogram of your data before training to check if it's normally distributed. Many algorithms assume normal distribution of errors. If data is very skewed, apply log transformation: np.log(data).

Z-Score

A z-score tells you how many standard deviations a value is from the mean. Standardises everything to the same scale.

Formula: z = (value − mean) / std_dev

z = 0 → exactly at the mean  |  z = 2 → 2 std devs above mean  |  |z| > 3 → likely outlier
PYTHON import numpy as np scores = np.array([55, 60, 72, 85, 91, 95, 30]) mean = np.mean(scores) std_dev = np.std(scores) z_scores = (scores - mean) / std_dev print(z_scores.round(2)) # Find outliers (|z| > 3) outliers = scores[np.abs(z_scores) > 3] print("Outliers:", outliers)
💡 Z-score standardisation IS what StandardScaler in sklearn does internally. Now you know exactly what's happening under the hood when you call StandardScaler().fit_transform(X).

IQR — Interquartile Range

IQR measures the spread of the middle 50% of your data. It's the most robust measure of spread because it completely ignores outliers.

Formula: IQR = Q3 − Q1

Q1 = 25th percentile (lower quarter)  |  Q3 = 75th percentile (upper quarter)

Outlier detection rule: any value below Q1 − 1.5×IQR or above Q3 + 1.5×IQR is an outlier. This is exactly what a box plot uses.
PYTHON import numpy as np data = [10, 12, 14, 15, 18, 20, 22, 100] # 100 is an outlier q1 = np.percentile(data, 25) # 12.5 q3 = np.percentile(data, 75) # 21.5 iqr = q3 - q1 # 9.0 lower = q1 - 1.5 * iqr # -1.0 upper = q3 + 1.5 * iqr # 35.0 outliers = [x for x in data if x < lower or x > upper] print("Outliers:", outliers) # [100]
🎯 Mean vs Median vs IQR: Use mean when data is clean. Use median when data has outliers. Use IQR to find and remove those outliers. In ML: always check IQR on your features before training — outliers can destroy linear models.

Correlation & Covariance

Correlation (Pearson) — measures if two variables move together. Always between −1 and +1.
+1 = perfect positive (as X goes up, Y goes up)

Basic Probability

Probability measures how likely an event is to happen. Always between 0 (impossible) and 1 (certain).

Formula: P(A) = favourable outcomes / total outcomes

P(Heads on coin flip) = 1/2 = 0.5
P(Rolling a 6) = 1/6 ≈ 0.167
P(Drawing a Heart from a deck) = 13/52 = 0.25

Complement rule: P(A does NOT happen) = 1 − P(A)

Conditional Probability

Conditional probability asks: "Given that B already happened, what is the probability of A?"

Formula: P(A|B) = P(A and B) / P(B)

Read P(A|B) as "probability of A given B".

Example: In a class of 40 students, 20 play cricket, 15 play football, 10 play both.
P(football | cricket) = P(both) / P(cricket) = (10/40) / (20/40) = 10/20 = 0.5
So 50% of cricket players also play football.
PYTHON — TITANIC CONDITIONAL PROBABILITY import pandas as pd df = pd.read_csv("titanic.csv") # P(Survived) overall print(df["Survived"].mean()) # 0.38 # P(Survived | Female) — survival rate of women print(df[df["Sex"] == "female"]["Survived"].mean()) # 0.74 # P(Survived | Male) print(df[df["Sex"] == "male"]["Survived"].mean()) # 0.19

Bayes' Theorem

Bayes' theorem lets you update your belief when you get new evidence.

Formula: P(A|B) = P(B|A) × P(A) / P(B)

Classic example — Medical Test:
Disease affects 1% of people. Test is 90% accurate (10% false positives). You test positive. What's the real probability you have the disease?

P(Disease) = 0.01   P(Positive | Disease) = 0.90   P(Positive | Healthy) = 0.10
P(Positive) = (0.90 × 0.01) + (0.10 × 0.99) = 0.108
P(Disease | Positive) = (0.90 × 0.01) / 0.108 = only 8.3%!

Even with a positive test for a rare disease, there is only an 8.3% chance you actually have it. This is why doctors do multiple tests.
PYTHON — BAYES FUNCTION def bayes(p_a, p_b_given_a, p_b_given_not_a): p_not_a = 1 - p_a p_b = (p_b_given_a * p_a) + (p_b_given_not_a * p_not_a) return (p_b_given_a * p_a) / p_b # Medical test print(bayes(0.01, 0.90, 0.10)) # 0.083 = 8.3% # Spam filter: P(spam)=0.3, P("free"|spam)=0.8, P("free"|not spam)=0.1 print(bayes(0.30, 0.80, 0.10)) # 0.774 = 77.4% chance it's spam
🎯 Naive Bayes classifier applies this formula to every word in an email. "Naive" = assumes each word is independent (not always true, but works well). You now understand the maths behind a real production ML algorithm used by Gmail.

Day 2 — Bayes Spam Classifier from Scratch

From the roadmap: build a real Naive Bayes spam filter using only Python — no sklearn. This is exactly how Gmail's spam filter works at its mathematical core.

The formula for each word:
P(spam | word) = P(word | spam) × P(spam) / P(word)
PYTHON — SPAM CLASSIFIER FROM SCRATCH # Tiny dataset: 1 = spam, 0 = not spam emails = [ ("free money now", 1), ("win free prize", 1), ("free offer today", 1), ("meeting at noon", 0), ("project update due", 0), ("call me today", 0), ] # Step 1: count words in spam vs not-spam spam_words = {} ham_words = {} spam_count = ham_count = 0 for text, label in emails: words = text.split() if label == 1: spam_count += 1 for w in words: spam_words[w] = spam_words.get(w, 0) + 1 else: ham_count += 1 for w in words: ham_words[w] = ham_words.get(w, 0) + 1 total = spam_count + ham_count p_spam = spam_count / total # prior: P(spam) = 0.5 p_ham = ham_count / total # prior: P(ham) = 0.5 # Step 2: classify a new email def classify(text): words = text.split() spam_score = p_spam ham_score = p_ham total_spam_words = sum(spam_words.values()) + 1 # +1 smoothing total_ham_words = sum(ham_words.values()) + 1 for w in words: spam_score *= (spam_words.get(w, 0) + 1) / total_spam_words ham_score *= (ham_words.get(w, 0) + 1) / total_ham_words return "SPAM" if spam_score > ham_score else "NOT SPAM" print(classify("free money offer")) # SPAM print(classify("call me today")) # NOT SPAM
💡 The +1 smoothing (Laplace smoothing) prevents the probability from becoming 0 when a word has never appeared in spam. If even one word has 0 probability, the entire product becomes 0 — so we add 1 to every count.

solved
Merge two sorted arrays into one sorted array in-place. Start from the end to avoid overwriting.
PYTHON def merge(nums1, m, nums2, n): i, j, k = m - 1, n - 1, m + n - 1 # start from the end! while i >= 0 and j >= 0: if nums1[i] > nums2[j]: nums1[k] = nums1[i]; i -= 1 else: nums1[k] = nums2[j]; j -= 1 k -= 1 while j >= 0: # remaining in nums2 nums1[k] = nums2[j]; j -= 1; k -= 1 # Pattern: Fill from the back — avoids shifting elements forward. # Always ask: can I solve this in O(n) with one pass?
